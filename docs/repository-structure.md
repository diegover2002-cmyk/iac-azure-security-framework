# Repository Structure

## Top-Level Layout

- `controls/`: Security control matrix, service catalogs, and detailed secure/insecure implementation examples.
- `docs/`: Canonical repository documentation for contributors and AI agents.
- `wiki/`: GitHub Wiki source pages.
- `scripts/`: Validation and reporting scripts used by CI/CD.
- `tests/terraform/`: Terraform examples used to validate secure and insecure patterns.

## Authoring Model

- Use `README.md` for the project overview.
- Use `AGENTS.md` and `Secure-Development-Guide.md` for contributor and agent rules.
- Use `docs/` for workflow, structure, and repository guidance.
- Use `wiki/` for navigable public-facing reference pages.
